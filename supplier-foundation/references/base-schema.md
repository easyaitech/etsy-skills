# 供应商管理 Base Schema

> 用 `lark-base` skill 在飞书云空间创建一份 Base：
> - `{店铺名}-供应商管理`
>
> Base 内核心表：
> - `采购来源`

---

## `采购来源` 表

### 核心字段（必建）

| 字段名 | 类型 | 说明 |
|---|---|---|
| `状态` | 单选 | `主用` / `备用` / `测试中` / `淘汰` |
| `物料名称` | 单行文本 | 采购对象名称，如纸盒、吊牌、丝带、陶瓷胚、包装袋 |
| `店铺名称` | 单行文本 | 供应商或平台店铺名称，按原平台展示名记录 |
| `商品链接` | 单行文本 | 供应商商品页链接；链接字段不稳定时先用文本保存 |
| `合适参数` | 多行文本 | 适用规格、尺寸、颜色、材质、起订量、价格区间等 |
| `选择理由` | 多行文本 | 为什么主用 / 备用 / 测试中；至少记录质量、稳定性、参数、交付或成本中的关键依据 |
| `淘汰原因` | 多行文本 | 状态为 `淘汰` 时必填，记录尺寸不符、质量不稳、价格不合适、交付慢等原因 |

### 状态语义

| 状态 | 用法 |
|---|---|
| `主用` | 当前首选来源，复购和日常采购优先用 |
| `备用` | 可替代来源，主用缺货、涨价或交付异常时切换 |
| `测试中` | 已发现但还没验证稳定性，需要小单测试或样品验证 |
| `淘汰` | 已验证不适合，保留原因防止重复选择 |

### 推荐视图

| 视图名 | 筛选 |
|---|---|
| `表格` | 全量，无筛选 |
| `主用` | 状态 = 主用 |
| `备用` | 状态 = 备用 |
| `测试中` | 状态 = 测试中 |
| `淘汰` | 状态 = 淘汰 |

## 创建命令参考

创建 Base：

```bash
lark-cli base +base-create \
  --name "{店铺名}-供应商管理" \
  --time-zone Asia/Shanghai
```

创建 `采购来源` 表时，字段最小 JSON：

```json
[
  {
    "name": "状态",
    "type": "select",
    "multiple": false,
    "options": [
      { "name": "主用", "hue": "Gray", "lightness": "Lighter" },
      { "name": "备用", "hue": "Green", "lightness": "Lighter" },
      { "name": "测试中", "hue": "Green", "lightness": "Lighter" },
      { "name": "淘汰", "hue": "Yellow", "lightness": "Lighter" }
    ]
  },
  { "name": "选择理由", "type": "text", "style": { "type": "plain" } },
  { "name": "淘汰原因", "type": "text", "style": { "type": "plain" } },
  { "name": "合适参数", "type": "text", "style": { "type": "plain" } },
  { "name": "物料名称", "type": "text", "style": { "type": "plain" } },
  { "name": "店铺名称", "type": "text", "style": { "type": "plain" } },
  { "name": "商品链接", "type": "text", "style": { "type": "plain" } }
]
```

状态视图创建后，用 `+view-set-filter` 设置：

```json
{ "logic": "and", "conditions": [["状态", "intersects", ["主用"]]] }
```

其他状态视图把 `主用` 替换为对应状态即可。

## 录入约定

1. 同一物料可以保留多个来源，不强行去重。
2. 同一 `物料名称 + 店铺名称 + 商品链接` 应视为同一来源，更新已有行。
3. `主用` 来源最好只保留少量真正可复购对象；其余放 `备用` 或 `测试中`。
4. 淘汰项不删除，除非是明显录错；保留淘汰原因比清空历史更有价值。
5. `商品链接` 失效时，不覆盖旧链接，先在 `选择理由` 或 `淘汰原因` 里记录“链接失效待复查”。

## 建表后的下一步

1. 给一个常用物料补 2-3 个来源，验证状态视图和筛选是否合用。
2. 把当前正在采购的物料先标为 `主用` 或 `测试中`。
3. 每次踩坑都补 `淘汰原因`，让这张表越用越聪明。
