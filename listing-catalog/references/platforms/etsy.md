# Etsy 平台 preset

> **⛔ 平台门槛**：本文件只在目标平台是 Etsy（`COMMERCE_PLATFORM.md` 明确选 Etsy，或用户目标明显是 Etsy）时读取。礼物 / 节日维度绑定 Etsy 的 listing 结构，**非 Etsy 平台不读本文件、不跑礼物 / 节日调研**。注册见 [`../../../shared/platform-config.md`](../../../shared/platform-config.md) § 内置平台 preset · Etsy。
>
> 本文件只写**长期有效的原则**。会随时间变的东西——具体热词、搜索量 / 竞争阈值、当年节日日期、第三方工具的界面——一律不写进来，由 agent 在写 listing 当时按实时数据判断。

---

## Etsy 是什么（决定一切的底层事实）

- **搜索驱动的礼物市场**：买家搜索意图大头是「给谁、什么场合送」，不只是搜品类。所以一条 listing 要同时回答「这是什么」和「谁会买、什么场合送」。
- **对结构化字段高度敏感**：Title / Tags / Materials / Category / Attributes 直接进 SEO 索引与筛选器，每个字段都是被搜到的机会。
- **礼物 / 节日是流量大头，是输入不是点缀**：礼物维度从输入端结构化收集（谁送、送谁、什么场合），驱动文案；不要写完文案再往后面贴节日 tag，那样会两层皮。

---

## 要填哪些参数 + 每个的总原则

| 参数 | 结构约束（长期不变） | 总原则 |
|---|---|---|
| **Title** | ≤ 140 字符，前段权重最高 | 品类核心词打头、含至少一个买家会搜的长尾词；礼物维度尽量靠前；语调随 BRAND.md；不堆砌、不空话（`best quality` / `100% authentic` 这类无 SEO 价值还显廉价） |
| **Tags** | 13 个，每个 ≤ 20 字符 | 全部填满；用词组不用单字；覆盖「品类 / 工艺材质 / 使用场景 / 礼物 / 风格」几个维度；不重复、不蹭品牌或 IP（会被下架） |
| **Materials** | 13 个 | 只填真实物理材质，不是把 tags 搬过来 |
| **Category** | 必选 | 选到最末级叶子节点；选错 → 错配受众、转化暴跌 |
| **Attributes** | 随类目变化 | 能填的全填，每填一个多一次被筛中机会；后台有固定枚举的项从选项里选，不自造新值 |
| **Description** | 无硬性字数 | 讲故事、不堆参数；开头几句最吃 SEO 与转化；政策口径引用 SHOP.md 原文、不自编；非自购品自然融入使用与礼物语境；不堆 SEO 词、不写空话（`perfect gift`）、不放 toB 词（`代发` / `OEM` / `批发`，这是 toC 平台） |
| **图 / 视频 Alt** | 每条 ≤ 125 字符 | 描述画面里看得到的东西、自然带 1-2 个核心词；多张图按主图 / 细节 / 场景差异化；兼顾无障碍与图搜，不堆词、不以 `Image of...` 开头 |

> 字段之间是同一个关键词来源在不同位置的复用：Title 放最值钱的几个词，Tags 把维度铺满，Materials 只承接材质，Description 把词织进故事。同一个词不要在多处机械重复。

## 礼物维度（Etsy 的核心输入）

写非自购品时，从输入端把礼物维度问清楚，构成一个「礼物词库」去驱动 Title 的礼物位、Tags 的礼物词、Description 的礼物语境：

- **先判断自购还是送礼**。自购为主的品不强加礼物词。
- **送礼则进一步明确**：送给谁（受众 / 关系）、什么场合、是否贴近某个节日。
- **投入与商品价值匹配**：低价 / 重复款轻量处理，高价值或主推品才深挖礼物角度——不要对每条 listing 都做满。
- **礼物词必须过 BRAND.md 语调过滤**：与品牌「避免说」冲突的、空泛商业感形容词（`perfect` / `luxury` / `adorable` 之类）一律不进词库；把被过滤掉的候选词列给用户看，以便反过来校正 BRAND.md。

### 附加调研环节：礼物维度调研（listing-catalog 模式 B step 5.5）

本 preset 在 listing-catalog 模式 B 的 step 5.5 声明一个附加调研环节，内容如下：

**(Etsy 专属，Etsy 下才跑；非 Etsy 平台整段跳过)** — **仅当目标平台是 Etsy 时执行**。礼物 / 节日维度是 Etsy 搜索流量大头、是 Etsy SEO 的输入，不套用到小红书或其他无礼物槽位的平台。注册见 [`../../../shared/platform-config.md`](../../../shared/platform-config.md) § 内置平台 preset · Etsy。按上文 § 礼物维度 的原则收集：先判断自购 / 送礼，送礼则明确受众 / 关系、场合、是否贴近节日；投入与商品价值匹配（低价轻量、高价值 / 主推品深挖）。

- 产出一个礼物词库（驱动 step 7 的 title 礼物位 / tags 礼物词 / description 礼物语境）+ 经 BRAND.md 语调过滤后的「过滤掉的候选词」清单
- **非 Etsy 平台**：跳过本环节——不收集礼物倾向、不产礼物词库，直接进 step 5.6

## 节日 / 季节

- **哪些节日重要**（西方主流礼物节日，长期稳定）：情人节、母亲节、父亲节、毕业季、婚礼季、万圣节、感恩节、圣诞、光明节、复活节。海外华人店可加春节、中秋。礼物属性弱的节日（如万圣节）只在品类契合时才追。
- **提前埋词的原则**：越靠近索引层的字段越早埋——Tags 最早（要给算法攒权重的时间）、Title 居中（太早把节日词放进可见文本会显廉价，比如夏天还挂圣诞）、Description 临门一脚（最贴近节日时再强化）。
- **具体日期每年变**，不写进本文件；上架前按当年日历和「距节日还有多久」自行判断词进哪个字段。节日 0 命中不打断流程。

---

## 上架前调研（工具无关）

- **动手前先验证**：这个角度有没有人搜、是不是红海、定价是否对得上市场（避免定在高价位却给不出「贵的理由」）。
- **怎么验证靠当时的工具**（eRank 等第三方）：用户在工具网页上查、把数据贴回来；本 skill 不爬取、不替登录。具体的搜索量 / 竞争阈值、工具界面都会变，不固化在本文件——按工具当时口径判断需求与竞争，按市场价格分布对标定价。
- **缺数据不硬等**：按 BRAND.md + 产品本身推断写出来，上线后看表现迭代。

## SEO 自检（原则版，每条写完过一遍）

- [ ] Category 选到叶子节点，Attributes 能填的全填
- [ ] Description 有故事、不堆参数，政策段引 SHOP.md 原文，非自购品有礼物 / 使用语境
- [ ] 整体符合 BRAND.md 语调，无空话与禁忌词
- [ ] 节日词按「提前期」原则进对的字段
- [ ] 每张图有差异化 alt，每条 listing 配视频且有视频 alt
