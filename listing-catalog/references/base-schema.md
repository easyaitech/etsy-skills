# 商品目录表推荐 Schema

> 用 `lark-base` skill 在店铺总 Base 内创建或补齐 `Products 商品` 表。**商品与 SKU 合并为一张表**：一行 = 一个可售 SKU，商品级故事 / 品类 / 分享链接与 SKU / 成本 / 售价 / 库存 / 平台商品 ID 同表共存，不再单独维护 `SKUs 变体` 表。
> 字段默认平台中性。历史 Etsy 工作区可以保留旧字段名，旧的独立 `Products` 与 `SKUs` 两表也合并进本表；新建表优先使用下面的通用字段；平台专属要求写入 `COMMERCE_PLATFORM.md`。
> 同一商品有多个变体时：简单情况用下方 `变体` 字段一行带过；变体多到需要各自定价 / 库存时，可为每个变体单独建一行（共享商品级字段），或另建变体子表关联。

## 核心字段（必建）

| 字段名 | 飞书字段类型 | 说明 / 示例 |
|---|---|---|
| `SKU` | 单行文本（**主键**） | 内部唯一编号，建议格式 `{品类缩写}-{数字}`，如 `TEACUP-001`、`POT-007`。一次建立，永不修改 |
| `销售平台` | 单选 / 多选 | Etsy / Amazon / Shopify / 淘宝 / 天猫 / 小红书店铺 / 抖店等；取值以 COMMERCE_PLATFORM.md 为准 |
| `平台商品 ID` | 单行文本 | 目标平台商品 ID，如 Etsy Listing ID、ASIN、Shopify handle、item_id；未上线为空 |
| `分享链接` | URL / 单行文本 | 平台后台或线上商品页复制出的可分享链接。Pinterest、小红书、Instagram、TikTok 等商品型发布必须优先读取本字段，不临时拼商品 URL |
| `状态` | 单选 | 草稿 / 待上线 / 在售 / 售罄 / 已下架 |
| `标题` | 多行文本 | 按 COMMERCE_PLATFORM.md 的目标平台语言与长度生成；Etsy preset 默认英文且 140 字符以内 |
| `描述` | 多行文本 | 商品描述原文，分段保留换行；输出语言由目标平台配置决定 |
| `关键词 / Tags` | 多行文本 / 单行文本 | 平台关键词、tags 或搜索词；数量、长度、分隔符按 COMMERCE_PLATFORM.md。Etsy preset 为 13 个 tag，用半角逗号分隔 |
| `材质 / 属性` | 多行文本 / 单行文本 | 平台要求的 materials、属性或卖点字段；Etsy preset 为 13 个 material，用半角逗号分隔 |
| `平台字段 JSON` | 多行文本 | 平台专属字段，如 Sustainability / Occasion / Holiday / Amazon bullet points / 淘宝属性等；只写有证据或已确认的字段 |
| `视频链接` | 文本 / 链接 | 商品页视频的飞书云盘链接或外链；是否必需、时长、比例按 COMMERCE_PLATFORM.md。原始视频走 `assets-library` |
| `售价` | 数字（货币） | 主售价；币种按 SHOP.md / COMMERCE_PLATFORM.md。变体定价用 `变体` 字段、单独变体行或平台字段 |
| `库存` | 数字 | 在售数量；定制款用 0 + 备注 "made to order" |
| `上线日` | 日期 | 状态 = 在售 时填，便于按时间查询 |

## 辅助字段（推荐，按需建，约 10 个）

| 字段名 | 飞书字段类型 | 说明 |
|---|---|---|
| `品类` | 单选 | 茶杯 / 茶壶 / 茶则 / 配套等 |
| `成本` | 数字（货币） | 估算成本，用于利润率计算；币种必须写清楚 |
| `利润率` | 公式 | `(售价 - 成本) / 售价`，自动算 |
| `制作周期(天)` | 数字 | 新订单平均制作周期（天） |
| `变体` | 多行文本或关联表 | 简单情况用文本（"Size: S/M/L; Glazed: Yes/No"）；变体需各自定价 / 库存时，每个变体单独建一行或另建变体子表关联 |
| `SEO 关键词` | 多行文本 | 当时为这条商品页想的关键词组（不全用上 tag，留作后续优化参考） |
| `照片链接` | 链接 / 附件 | 飞书云空间该 SKU 摄影目录链接（assets 库 skill 维护） |
| `图片说明 / Alt` | 多行文本 | 平台支持 alt text 或图片说明时填写；长度和语言按 COMMERCE_PLATFORM.md。Etsy preset 每行 ≤ 125 字符 |
| `视频说明 / Alt` | 多行文本 | 平台支持视频 alt 或说明时填写；长度和语言按 COMMERCE_PLATFORM.md |
| `备注` | 多行文本 | 任何特殊情况：定制规则、容易碎、不发某些国家、限定批次等 |

## 小红书字段（启用小红书平台时必建）

> 依据 `references/platforms/xiaohongshu.md`。这些字段服务小红书 SPU / SPL / SPV / ITEM 分层。字段可以先为空，但建表时要预留，避免后续上新时把平台信息塞进备注。

### 小红书 SPU / 类目 / 品牌

| 字段名 | 飞书字段类型 | 说明 |
|---|---|---|
| `小红书商品 ID` | 单行文本 | 小红书 ITEM 商品 ID；上线后从后台或 API 回填 |
| `小红书 SPU ID` | 单行文本 | SPU ID；创建或导入后回填 |
| `小红书 SPL ID` | 单行文本 | SPL ID；同一展示规格共享图片和描述 |
| `小红书 SPV ID` | 单行文本 | SPV ID；具体可下单规格 |
| `小红书品牌 ID` | 单行文本 | `brand_id`；先从品牌搜索 / 后台品牌库确认 |
| `小红书品牌名` | 单行文本 | 人读字段，便于核对品牌 ID |
| `小红书末级类目 ID` | 单行文本 | `category_id`，必须是最末级分类 ID |
| `小红书类目路径` | 多行文本 | 一级到末级类目路径，便于人工复核 |
| `商品英文名` | 单行文本 | `ename`；没有英文名时留空 |
| `商品简称` | 单行文本 | `short_name`，不超过 15 个中文字符 |

### 小红书文描 / 媒体

| 字段名 | 飞书字段类型 | 说明 |
|---|---|---|
| `小红书商品特色` | 单行文本 | `feature`，不超过 8 个字符 |
| `小红书商品图链接` | 多行文本 | SPL_ITEM `image_urls`；一行一个图片链接 |
| `小红书使用指南图链接` | 多行文本 | `user_guide.image_urls`；一行一个图片链接 |
| `小红书图文详情图链接` | 多行文本 | `image_desc.image_urls`；一行一个图片链接 |
| `小红书产品参数 JSON` | 多行文本 | `attributes`，保存 `property_id` / `value_id` / `value` |
| `小红书 FAQ JSON` | 多行文本 | `faqs`，保存 question / answer |
| `小红书审核状态` | 单选 | 待补资料 / 待提交审核 / 审核中 / 审核通过 / 审核驳回 |
| `小红书审核备注` | 多行文本 | 记录驳回原因、待补字段和人工确认结果 |

### 小红书规格 / SKU / 库存

| 字段名 | 飞书字段类型 | 说明 |
|---|---|---|
| `小红书 SPL 规格 JSON` | 多行文本 | `variants`，保存规格 ID、规格值、值 ID |
| `小红书 SPV 规格 JSON` | 多行文本 | `non_desc_variants`，保存尺码 / 颜色 / 套装等 SKU 规格 |
| `小红书 SKU 编码` | 单行文本 | `barcode` 或后台商品编码；只能输入字母、数字、`/`、`-` 时按平台要求校验 |
| `小红书条码类型` | 单选 | UPC / EAN / 自定义 / 待确认 |
| `小红书 SKU 状态` | 单选 | 未启用 / 已启用 / 已下架 / 待确认 |
| `小红书库存` | 数字 | SKU 维度库存；不要只维护 SPU 总库存 |
| `小红书上下架状态` | 单选 | 草稿 / 上架 / 下架 / 不可售 / 待确认 |

### 小红书价格 / 商品基础信息

| 字段名 | 飞书字段类型 | 说明 |
|---|---|---|
| `小红书售价(CNY)` | 数字（货币） | ITEM `price`；非跨境税商家必填 |
| `小红书原价(CNY)` | 数字（货币） | ITEM `original_price` |
| `小红书税前价(CNY)` | 数字（货币） | ITEM `pre_tax_price`；跨境税商家必填 |
| `小红书内含数量` | 数字 | SPV `qty`，注意不是库存 |
| `小红书单位` | 单行文本 | SPV `unit`，如 件 / 套 / ml / g |
| `小红书净重(g)` | 数字 | SPV `net_weight` |
| `小红书毛重(g)` | 数字 | SPV `gross_weight` |
| `小红书原产国` | 单行文本 | SPV `country` |
| `小红书保质期(天)` | 数字 | SPV `shelf_life`；无保质期商品可留空 |
| `小红书生产厂家` | 单行文本 | SPV `manufacturer`，填写全称 |
| `小红书材质/成分` | 多行文本 | SPV `ingredient` |
| `小红书用途` | 多行文本 | SPV `usage` |
| `小红书规格型号` | 多行文本 | SPV `customs_specification` |
| `小红书海关备案图链接` | 多行文本 | `customs_photos_urls`，跨境或需备案时填写 |
| `小红书物流模式` | 单行文本 | 默认物流模式或后台选择结果；不确定时写待确认 |

## 进阶字段（可选，等业务跑顺再加）

- `首次纠正记录` / `last_correction`：记录用户最近一次对该 listing 的纠正反馈（用于回流到 BRAND.md 沉淀）
- `引用的品牌原则`：多选关联（如哪些 BRAND.md 原则是该 listing 重点体现的）—— 适合做品牌一致性 audit 时追溯
- `合规标记`：是否含定制/IP/限定地区物料（影响平台政策合规）

## 视图建议（建完字段后顺手建）

- **全部** — 默认视图
- **在售** — 状态=在售，按上线日倒序
- **草稿与待上线** — 状态∈{草稿, 待上线}，按 SKU 升序
- **缺货** — 库存=0 且状态=在售
- **按品类** — 分组视图

## 字段命名约定

- 中文标签优先（用户用中文交互）；平台后台有固定英文名时可把原名写进字段说明或 `平台字段 JSON`，不要为了某个平台把通用 Base 主字段全部改成平台专用名
- 主键 `SKU` 用大写英文 + 数字，方便 grep 和不同表关联

## 不建议在 Base 里塞的东西

- 摄影原图：体积大、版本多 → 走资产库（`assets-library` skill）+ 链接字段
- 客户订单数据：店铺总 Base 内 `Orders 订单` / `Customers 客户` 表（`orders-customers` skill）
- 评价记录：店铺总 Base 内订单 / 客户相关表
- 长篇 listing 描述的草稿历史：写完确认就覆盖；版本控制不该在 Base 里做

## 反查素材：商品表 ↔ `Assets 素材池` 表

商品表不直接存放图片文件本体——素材通过 `Assets 素材池` 表反向关联（关联 SKU 字段）。在飞书 Base 里点开某 SKU 行，能看到所有指向它的素材行。

**注意范围**：反查到的是该 SKU 的 **promoted 成品**（已经走过 assets-library 模式 B2 的）。**未录入 Base 的素材不在反查结果里**——找未分类素材浏览 `素材库/待处理/` 文件夹（详见 [assets-library/references/folder-structure.md](../../assets-library/references/folder-structure.md)）。

是否在商品表上加"照片链接"快捷字段，由 assets-library 的 [asset-types.md § 与 listing-catalog 的协作](../../assets-library/references/asset-types.md#与-listing-catalog-的协作) 在每次 promote 时反向问用户，本表不预设。

## 建表后的下一步

1. 让用户给一个示范 SKU（已上线的或一个准备上的），手填一行让你看 schema 是不是合用
2. 如果合用：可以批量从平台后台或现有 Excel 导入历史 SKU（用户自己导，本 skill 不替操作平台后台）
3. 用户用一两周后回头补辅助字段
