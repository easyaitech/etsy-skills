# Scan And Dedupe

本流程定义如何从飞书文件夹 / 云文档发现新素材，并登记到素材池。它只描述流程；真实扫描、下载、写 Base 都必须经用户确认。

---

## 输入

用户可能提供：

- 飞书云盘文件夹链接
- 飞书云文档链接
- 聊天上传的一批素材
- `Products 商品` / `SKUs 变体` 表中引用的素材链接

扫描前先确认目标：

- 只是盘点，不写 Base
- 新素材写入素材池
- 为某个平台准备发布任务

---

## 扫描步骤

1. 从用户给的飞书文件夹 / 云文档提取 token。
2. 列出文件。
3. 对每个文件提取：
   - file token
   - 文件名
   - 文件链接
   - 文件类型
   - 修改时间
4. 下载到 workspace cache，用于计算 hash 和后续发布副本处理。
5. 计算内容 hash。
6. 用 `file token + hash` 判断是否已登记。
7. 已登记则跳过。
8. 新素材写入素材池，状态为 `待处理` 或 `待清理`。
9. 不直接发布。
10. 不删除、移动或覆盖原图。

建议 cache 路径：

```text
<workspace>/.cache/content-asset-pool/incoming/{YYYYMMDD}/
```

---

## 去重规则

### 一级：file token

如果 `原图 file token` 已存在：

- 视为同一个飞书文件。
- 不新增素材记录。
- 可追加备注：`duplicate scan: seen at <timestamp>`。

### 二级：hash

如果 file token 不同，但 `原图 hash` 已存在：

- 视为内容重复或重新上传副本。
- 不自动合并。
- 标记 `需人工复核`，让用户决定是否关联到同一素材或保留为独立来源。

建议备注：

```text
same hash as ASSET-20260524-001, different file token; needs human review
```

### 三级：文件名相似

文件名相似但 token 和 hash 都不同，不视为重复。只在备注中提示相似候选，避免误删或误合并。

---

## 新素材初始状态

图片 / 视频只要还没判断授权、SKU、用途，初始：

```text
素材生命周期状态 = 待处理
AI 清理状态 = 未处理
像素级水印处理 = 未处理
授权状态 = 待确认
```

如果来源已知为自有素材，且用户明确要用于发布，但还没生成发布副本：

```text
素材生命周期状态 = 待清理
授权状态 = 自有
AI 清理状态 = 未处理
```

如果来源是客户 UGC 或外部素材：

```text
授权状态 = 待确认
素材生命周期状态 = 待处理
```

未授权素材不得进入 Publishing Queue。

## 写入前预览

写入素材池前必须展示：

```text
将新增素材：
1. ASSET-20260524-001
   名称: ...
   类型: 图片
   来源链接: ...
   file token: ...
   hash: ...
   初始状态: 待处理

将跳过重复：
1. ...

需人工复核：
1. ...
```

只有用户明确确认后才用 `lark-base` 写入。

---

## 禁止事项

- 不把云盘文件夹当发布状态系统。
- 不通过移动文件夹表达“已发布 / 已入队”。
- 不删除重复文件。
- 不覆盖原图。
- 不把同一素材复制到多个文件夹表达多用途。
- 不在扫描流程里创建 Pinterest `Pinterest Queue` 表或平台发布任务；发布任务必须由用户确认后另走 Workflow D。
