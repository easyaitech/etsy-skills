# State Model

素材状态和发布状态必须分开。素材池描述“素材是否准备好”；发布任务描述“某个平台的一次发布是否完成”。

---

## 素材生命周期状态

推荐状态：

```text
待处理
  ↓
待清理
  ↓
可发布
  ↓
已入任务
  ↓
可复用 / 停用 / 不可公开
```

### 状态定义

| 状态 | 含义 | 允许动作 |
|---|---|---|
| `待处理` | 已发现或登记，但还没判断授权、类型、SKU、用途 | 补字段、查来源、人工判断 |
| `待清理` | 可以作为候选，但发布副本还没完成 metadata/provenance 清理 | 生成发布副本 |
| `可发布` | 授权允许，发布副本就绪，适合进入发布任务 | 创建 Publishing Queue 草稿 |
| `已入任务` | 已被一个或多个发布任务引用 | 等待下游发布、继续复用 |
| `可复用` | 已完成至少一次任务，但仍允许用于其他平台或后续内容 | 创建新平台任务 |
| `停用` | 暂不使用，但不代表不可公开 | 不创建新任务，保留历史 |
| `不可公开` | 授权或内容风险导致不能公开发布 | 阻塞所有发布任务 |

### 关键规则

- 素材池不要写简单的 `已发布`。同一素材可能 Pinterest 已发，小红书未发，Instagram 还没发。
- `已入任务` 只说明已经进入某个发布任务，不说明发布成功。
- 已发布后是否改为 `可复用` 由人工策略决定；不要自动停用。
- `不可公开` 优先级最高。只要授权状态或内容风险不可公开，不能因为已有发布副本而进入任务。

---

## 发布任务状态

基础路径：

```text
草稿
  ↓
待发
  ↓
发布中
  ↓
已发
```

失败路径：

```text
待发 / 重试
  ↓
发布中
  ↓
失败
  ↓
待复核 / 重试 / 跳过
```

### 状态定义

| 状态 | 含义 | 允许动作 |
|---|---|---|
| `草稿` | 素材、文案、链接、平台已初步确定 | 编辑、确认、补字段 |
| `待发` | 已通过人工确认，等待平台发布 skill 执行 | 发布、排期 |
| `发布中` | `social-publisher` 已占用任务并写入 `执行锁`，正在调用平台适配器 | 等待结果回写；中断时先核对平台真实结果 |
| `已发` | 平台发布成功，已写 `发布 URL` 和 `发布时间` | 对账、复盘 |
| `失败` | 平台发布动作失败 | 写失败原因，决定复核或重试 |
| `待复核` | 需要人工判断是否改内容、换素材、重试 | 人工处理 |
| `重试` | 允许再次进入待发路径 | 回到 `待发` |
| `跳过` | 本任务不再发布 | 保留记录，不删除 |

### 发布任务对素材池的回写

- 创建任务后：素材池 `关联发布任务` 追加任务 ID，`素材生命周期状态` 可改为 `已入任务`。
- 发布成功后：只更新发布任务状态并清空 `执行锁`，不把素材池改成 `已发布`。
- 如果用户确认该素材仍可跨平台复用，可把素材池状态改为 `可复用`。
- 发布失败不会自动让素材变成不可公开；失败原因属于发布任务，除非失败暴露授权或内容风险。
- `发布中` 是短暂执行状态。若工具中断导致任务停在 `发布中`，必须先核对平台真实结果，再改为 `已发` / `失败` / `待复核`；不能直接再次发布。
