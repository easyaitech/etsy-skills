# Publishing Queue Contract

social-publisher 消费 `content-asset-pool` 创建的 `{店铺名}-发布任务` 表。发布状态只以这张表为跨平台 source of truth；平台子队列（如 Pin Queue）是适配器执行细节。

## 必填字段

| 字段 | 说明 |
|---|---|
| `任务 ID` | `PIN-...` / `XHS-...` / `IG-...` 等 |
| `平台` | Pinterest / 小红书 / Instagram / TikTok / Etsy |
| `发布类型` | 单图 / 多图轮播 / 视频 / 图文笔记 / 图文混合 |
| `关联素材` | 一条或多条素材记录 |
| `素材顺序` | 多素材任务的唯一顺序来源 |
| `封面素材` | 多图 / 视频 / 图文笔记必须填 |
| `标题` | 平台标题 |
| `描述` | 平台正文 / caption |
| `链接` | 商品型发布必须来自商品 Base `分享链接`；非商品型可空 |
| `状态` | 发布状态 |
| `计划发布时间` | 自动发布筛选依据 |
| `自动发布` | true 才允许无人值守自动执行 |

## 发布器字段

| 字段 | 类型建议 | 说明 |
|---|---|---|
| `发布适配器` | 单选 / 文本 | 如 `pinterest-autopin`、`manual-xiaohongshu` |
| `外部队列 ID` | 单行文本 | 平台子队列 ID；Pinterest 填 Pin Queue `pin_id` |
| `发布尝试次数` | 数字 | 每次进入真实发布前累加；默认 0 |
| `最后尝试时间` | 日期时间 | 每次自动 / 手动执行后更新 |
| `执行锁` | 单行文本 | 真实发布前写入本轮唯一令牌；完成、失败或放弃后清空 |

## 状态机

```text
草稿
  └─ 人工确认排期 → 待发
待发
  ├─ 到点 + 自动发布=true + adapter enabled → 发布中
  ├─ 手动执行 → 发布中
  └─ adapter unavailable / 资料缺失 → 待复核
发布中
  ├─ 成功 → 已发
  └─ 失败 → 失败
失败
  ├─ 用户确认重试 → 重试
  ├─ 人工处理 → 待复核
  └─ 放弃 → 跳过
重试
  ├─ 到点或手动执行 → 发布中
  └─ 达到重试上限 → 待复核
```

## 占用规则

真实调用平台适配器前，必须先占用 Publishing Queue 行：

1. 为本轮生成唯一 `执行锁`，例如 `social-publisher:<run_id>:<任务 ID>`。
2. 使用条件更新或等价机制，只在当前行仍是 `待发` / `重试` / 用户确认的 `草稿`，且 `执行锁` 为空时，写入 `状态 = 发布中`、`执行锁`、`最后尝试时间 = 当前时间`、`发布尝试次数 +1`。
3. 重新读取该行，确认 `状态 = 发布中` 且 `执行锁` 等于本轮令牌；不成立就停止，不调用 adapter。
4. 如果当前工具无法做条件更新，也无法证明唯一占用，自动巡检只能单实例运行；不能并发启动多个 runtime。
5. 成功、失败、待复核或跳过后清空 `执行锁`。中断遗留的 `发布中` 任务，必须先核对平台真实结果，再改成 `已发` / `失败` / `待复核`，不能直接再次发布。

## 自动发布筛选

自动巡检只能选择同时满足以下条件的记录：

- `自动发布 = true`
- `状态 = 待发` 或 `状态 = 重试`
- `计划发布时间 <= 当前时间`
- `发布尝试次数 < 2`
- 平台 adapter 在 `adapter-registry.md` 中是 `enabled`

每个平台每轮默认最多发布一条，按 `计划发布时间` 最早优先。

## Pinterest 映射

Pinterest Publishing Queue 行必须映射到 Pin Queue：

| Publishing Queue | Pin Queue |
|---|---|
| `任务 ID` | `pin_id`，可复用 `PIN-...` |
| `发布类型` | `pin 类型`：单图 / 轮播 |
| `关联 SKU` | `关联 SKU` |
| `关联素材` | `关联素材` |
| `素材顺序` | `image 路径` 的行顺序 |
| `标题` | `Title (EN)` |
| `描述` | `Description (EN)` |
| `链接` | `Link` |
| `平台字段 JSON.board` 或人工字段 | `Board (Pinterest)` |

发布成功后：

- Pin Queue：`状态 = 已发`、`pin_url`、`发布时间`
- Publishing Queue：`状态 = 已发`、`发布 URL`、`发布时间`、`外部队列 ID`

发布失败后两边都要记录失败，不允许只改 Pin Queue。

## Manual-only 平台

小红书、Instagram、TikTok 在 adapter 启用前只能：

- 生成 Publishing Queue 草稿
- 输出人工发布清单
- 用户给出公开 URL 后回写 `已发`

不得自动登录、上传、点击发布，也不得把“草稿已准备”写成“已发”。
