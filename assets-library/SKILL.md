# 资产库 (Assets Library)

这个 skill 维护 Etsy 店铺的视觉与素材资产，核心是**结构化目录 + 一致命名 + 与商品 / Listing 的可追溯链接**。

**对外的实操接口**：飞书云空间（用 `lark-drive` skill 操作）+ 项目根目录的 BRAND.md（视觉原则）。

---

## 依赖关系

| 来源 | 提供什么 | 怎么用 |
|---|---|---|
| `./BRAND.md` § 视觉原则 | 整体气质 / 色彩 / 排版构图 / 视觉禁区 | 归档新摄影 / 处理新模板时用作"是否合规"自检 |
| 商品 Base | SKU 列表 | 摄影目录按 SKU 归档；Base 的"照片链接"字段填飞书云空间链接 |

---

## 三种执行模式

### 模式 A：建库（首次建立目录骨架）

**进入条件**：飞书云空间还没有规范的资产目录。

**执行步骤**：
1. 读 `references/folder-structure.md`，了解推荐目录骨架
2. 用 lark-drive skill 在飞书云空间创建根文件夹 `{店铺名}-素材库`
3. 按推荐结构创建子目录（摄影 / 视觉 / 字体 / Logo / Listing 配图 / 临时）
4. 落盘后告诉用户根目录链接 + 子目录清单 + 简要使用约定（命名规则、上传规则）

> **不要替用户上传任何文件**——本 skill 在建库阶段只创建空骨架。让用户自己把已有素材分批迁过来；迁的过程中可以求助本 skill 给命名建议。

### 模式 B：归档新素材

**进入条件**：用户上传了新摄影 / 设计 / 模板，需要按规范归档。

**执行步骤**：
1. 读 `references/naming-convention.md`：确认命名规则
2. 读 `references/asset-types.md`：识别素材类型 → 应该归到哪个子目录
3. 给出**目标路径 + 标准化文件名**建议：
   ```
   素材库/摄影/by-SKU/TEACUP-001/raw/2026-05-05_TEACUP-001_morning-light_001.jpg
   ```
4. 用户确认后，用 lark-drive 创建子目录（如缺）+ 上传 / 移动文件
5. 如果是按 SKU 归档的摄影：建议用户回到商品 Base 该 SKU 行更新"照片链接"字段（由 listing-catalog 完成）

> **视觉合规检查**：归档前如果素材明显违反 BRAND.md 视觉禁区（如刻板符号、高饱和、AI 拼贴感），主动提示——但**不替用户拒绝归档**，只负责提醒；最终决策权在用户。

### 模式 C：查找

**进入条件**：用户要找某个素材（"某 SKU 的图"、"上次那个新春模板"、"Logo PNG 透明底"）。

**执行步骤**：
直接调用 `lark-drive` skill 的搜索功能。本 skill 不重复实现检索，仅在以下场景介入：
- 用户描述模糊时引用 `references/folder-structure.md` 推断可能位置
- 涉及多个候选时，按命名约定排序提供链接列表

---

## 写入前的硬性约束

1. **文件操作**：上传 / 移动 / 重命名 / 删除前，列出动作清单，**等用户确认**才执行
2. **不删除原始素材**：摄影原图（RAW / 高清 JPG）一旦归档，本 skill 不主动删除——容量管理由用户人工决策
3. **不替用户处理图片内容**：本 skill 只管**目录、命名、归档**；图片裁切、调色、加水印、生成场景图等由用户人工处理或调用其他工具
4. **隐私素材**：客户照片、定制订单的隐私材料（如客户提供的家庭照片用于定制礼物）单独建受限子目录，不与公开摄影混放

---

## 与其他 skill 的协作

- **shop-foundation**：每次归档后如果发现 BRAND.md 视觉原则需要补充（比如新增了"自然光必拍"这种偏好），按 shop-foundation 沉淀流程提议进 BRAND.md
- **listing-catalog**：摄影按 SKU 归档后，回到商品 Base 更新"照片链接"字段——由 listing-catalog 完成
- **orders-customers**：客户提供的定制图归到"定制素材"受限目录，与订单 Base 的对应订单关联

---

## 工作语言

- 与用户对话：中文
- 文件名：英文（避免飞书云空间路径里有中文带来的兼容问题，且便于跨平台使用）
- 目录名：可中英混用（飞书原生支持中文目录名）
