# Etsy Listing 图位社区 SOP

> ⚠️ **平台 vs 社区惯例**：Etsy 平台只规定 listing 最多 **10 张图 + 1 段视频**，槽位**没有平台级语义**——任何顺序、任何内容都能上传。
>
> 本文档列出的 **hero / variation / scale / size chart / detail / lifestyle / packaging / brand story / context / comparison** 是**社区最佳实践的语义映射**，不是 Etsy 平台规则。Agent 引用本文档时，应明确告诉用户"这是社区惯例 SOP，不是平台强制"——避免用户误以为 Etsy 后台真有这些槽位标签。
>
> Agent 跟用户表述时建议用"建议槽位 1 / 2 / 3..."而非"hero / variation slot"等英文术语，避免给用户造成"Etsy 平台真有这些命名"的错觉。

image-brief 用本文档的槽位定义生成 shoot brief 的 §A 表（Etsy 10 槽位映射表）+ §C 镜头清单。

---

## 1. 优先级定义

| 优先级 | 含义 | 缺失影响 |
|---|---|---|
| **P0** | listing 上线缺则不达标 | 搜索结果 / 列表页主图位空，转化率塌方；不能上线 |
| **P1** | 高优先级，几乎所有品类必有 | 详情页转化下降；用户无法判断尺寸 / 材质 / 工艺 |
| **P2** | 场景 / 包装类，按品类决定必拍性 | 礼物搜索意图捕获不足；售后退货率上升（不知道实物多大） |
| **P3** | 可选锦上添花 | 不影响达标，但缺则品牌叙事单薄 |

**与 §A 表的契约**：每个槽位的"必拍?"列由优先级直接派生：P0 / P1 = ✅必拍；P2 = ✅按品类；P3 = 可选。

## 2. 10 槽位社区语义

### 槽位 1 — Hero（搜索结果展示）

- **优先级**：P0
- **作用**：Etsy 搜索结果页 + 类目列表页 + 收藏夹 thumbnail 用的就是这张；listing 详情页打开后第一张也是它。整个 listing 的"封面"。
- **典型构图**：单品居中 + 留白；白底或品牌纯色底；正面 / 侧 45° / 俯视三选一
- **典型尺寸**：Etsy 推荐主图 **2700 × 2025 px**（4:3）；竖图也行但搜索结果会被裁切
- **常见错误**：
  - 用 lifestyle 场景图当 hero（场景图细节多、缩略图缩成豆腐块看不清）
  - 主体不居中（被搜索结果 thumbnail 自动裁切）
  - 用了水印 / 价格标签（违反 Etsy 政策；竞争力下降）

### 槽位 2 — Variation grid（变体集合）

- **优先级**：P0（仅当 SKU 有变体时）
- **作用**：颜色 / 尺寸 / 款式变体的整体展示；让用户一眼看到全部选择
- **典型构图**：网格平铺 / 横向并排；统一光线和角度
- **常见错误**：
  - 不同变体光线 / 背景不一致（看起来像不同店）
  - 漏掉某个变体（用户在 variation 下拉里看到但图里没有）
  - 单 SKU 也硬塞 variation grid（占图位浪费）

### 槽位 3 — Scale（尺度感）

- **优先级**：P1
- **作用**：让用户判断实物多大；是 Etsy 退货第一大原因（"比想象的小 / 大"）的解药
- **典型构图**：商品 + 参照物（手 / 硬币 / 信用卡 / 已知尺寸的家具）；同框拍
- **常见错误**：
  - 用抽象参照物（"和我手掌差不多"——每个人手不一样大）
  - 参照物不是国际通用（用人民币硬币 → 美国买家不知道有多大）
  - 单独用尺子拍（不够直观，做成 size chart 更合适）

### 槽位 4 — Size chart（规格表）

- **优先级**：P1（功能性 / apparel SKU 必拍；纯装饰品可选）
- **作用**：精确尺寸数据；apparel 品类 Etsy 会专门要求
- **典型构图**：商品平铺 + 标尺标注 / 表格图层 / Photoshop 数字标注
- **常见错误**：
  - 只标 cm 不标 inch（北美买家主流 inch）
  - 标注字小到看不清
  - 跟 Scale 重复（Scale 给直觉，Size chart 给精确数据，二者不能互相替代）

### 槽位 5–7 — Detail（细节特写 ×3）

- **优先级**：P1（至少拍 3 张：材质 / 工艺 / 关键纹理或 Logo）
- **作用**：手工 / 高客单价品类的转化关键；Etsy 用户买的是"工艺"和"独特性"
- **典型构图**：微距特写；柔和光突出纹理；浅景深虚化背景
- **常见错误**：
  - 3 张都拍同一个细节角度（应该轮换：材质 / 工艺 / Logo）
  - 微距拍但对焦没对到关键点
  - 只拍美的细节，回避瑕疵——用户收到货发现有瑕疵 → 退货 + 差评

### 槽位 8 — Lifestyle（场景图）

- **优先级**：P2（强烈推荐）
- **作用**：礼物搜索意图捕获；buyer 投射"用这个的我"
- **典型构图**：人 + 物 / 物 + 使用情境；自然光；故事性构图
- **与 listing-catalog 礼物词库的关系**：场景词库（受众 / 场景 / 节日 / 包装）每条都对应一个 lifestyle shot
- **常见错误**：
  - 场景跟 listing description 礼物叙事不同步（描述讲春天感、图全是冬天）
  - 用模特但模特抢戏（应该 70% 商品 + 30% 模特）
  - 摆拍痕迹太重（"假笑模特拿杯子"——比无 lifestyle 还差）

### 槽位 9 — Packaging（包装 / 拆箱）

- **优先级**：P2
- **作用**：礼物送礼场景必备；建立"高级感"和品牌差异
- **典型构图**：包装盒 + 商品半露出 / 拆箱过程定格 / 卡片 + 商品
- **与 SHOP.md 物料的关系**：拍 SHOP.md `§ 物料` 列出的实际物料（包装盒 / 感谢卡 / 贴纸 / 礼盒服务）
- **常见错误**：
  - SHOP.md 没配置物料但硬拍 packaging（拍出来跟实际包装不一致 → 用户收到失望）
  - 拍空盒子没商品（应该展示包装 + 商品的关系）
  - 包装跟品牌风格不统一（廉价快递盒装高客单价手作 → 品牌一致性塌方）

### 槽位 10 — Brand story / context（品牌故事 / 工坊环境）

- **优先级**：P3（可选）
- **作用**：差异化 + 信任建立；让品牌不只是"商品"而是"故事"
- **典型构图**：制作过程 / 工坊环境 / 创作者照片 / 原料展示
- **与 BRAND.md 的关系**：取 BRAND.md `§ 起源 / § 工艺` 段的视觉化呈现
- **常见错误**：
  - 用 stock 图（违背手作精神，反向破坏信任）
  - 故事跟 BRAND.md 叙事不一致（"匠人工坊"+ "工业化车间"图）
  - 槽位 10 占了，但 P0/P1 还有空（颠倒优先级）

## 3. 槽位 ID 与 `Assets 素材池` 表 "用途标签" 字段对齐

image-brief 部分跑（老 SKU 补拍）会用 lark-base 反查 `Assets 素材池` 表中该 SKU 已 promoted 素材的"用途标签"，推断已覆盖槽位。这要求两边的词汇表对齐。

**契约词汇表**（`Assets 素材池` 表 "用途标签"字段必须包含；建库 / 升级时同步加入）：

| 槽位 | 槽位 ID（"用途标签"字段值）|
|---|---|
| 1 | `hero` |
| 2 | `variation` |
| 3 | `scale` |
| 4 | `size-chart` |
| 5–7 | `detail` |
| 8 | `lifestyle` |
| 9 | `packaging` |
| 10 | `brand-story` |
| — | `context`（社区惯例，本 SOP 未单独列槽位但保留 ID）|
| — | `comparison`（对比图，少见但保留）|

> 当前 `Assets 素材池` 表的"用途标签"字段也用于渠道标记（`Pinterest / Instagram / 小红书 / 评价素材` 等）——本文档的槽位 ID 与渠道标签**共用同一个多选字段**，不冲突。一个素材可同时勾 `hero + Pinterest`（既是 listing 主图又投 Pinterest）。

**降级路径**：如果 `Assets 素材池` 表当前 schema "用途标签"字段词汇表里**没有**槽位 ID（即用户库存量大、历史标签词汇表是 freeform 的），image-brief 部分跑分支降级为"让用户口头列已覆盖槽位"（不阻塞），并提示用户后续在 schema 升级时加上槽位 ID 词汇。

## 4. PR1 范围说明

PR1 仅给**通用 10 槽位 SOP**——本文档当前内容。

按品类细化（apparel / jewelry / home & living / digital / paper goods）的优先级矩阵留作 v2，等image-brief 至少跑过 1 次实战后再决定颗粒度——避免 over-engineering 出与实际拍摄场景不匹配的细化规则。

---

## 5. Etsy SEO 与图位的连接

title / tag 里的关键词应该在视觉里能看到。**反向自检**：

- 主图（hero）必须呈现 title 第 1-3 关键词暗示的视觉（搜 "morning tea cup" 命中 → hero 必须有 morning + tea + cup 元素）
- description 的礼物语境必须有 lifestyle 槽位对应图（讲新婚礼物 → 必须有伴娘场景图）
- 礼物词库各类（受众 / 场景 / 节日 / 包装）每类至少有 1 张视觉对应图

如果 listing-catalog 模式 B 已生成文案而 shoot brief 还没补足这些视觉对应——**image-brief 应该提示用户**："你 listing 讲了 X 但 brief 里没对应 lifestyle 拍摄计划，要补吗？"
