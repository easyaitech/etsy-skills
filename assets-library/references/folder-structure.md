# 推荐目录骨架

> 在飞书云空间创建 `{店铺名}-素材库` 作为根目录，下面按以下结构铺开。

## 文件夹的定位（很重要）

文件夹是**物理位置**，只负责回答"这个文件物理放在哪"。
"它属于哪些 SKU / 渠道 / 客户 / 用途"由[素材索引 Base](asset-index-base-schema.md) 通过多选字段和关联字段表达——**不要试图用嵌套目录表达多维归类**。

例如"客户拍的、能作 Pinterest 评价图的茶杯特写"——这条素材：
- 物理位置：`7. 受限/客户拍摄/{ORDER-XXXX}/`（客户原物的唯一物理位置）
- 多维归类：在素材索引 Base 一行里 `素材类型=[摄影成图, 客户拍摄]` + `关联订单=ORDER-XXXX` + `用途标签=[Pinterest, 评价素材]` + `公开授权=已授权`

绝不要为这条素材另建 `客户评价图/Pinterest候选/` 之类的目录，也不要把同一文件复制到多处。

```
{店铺名}-素材库/
├── 1. 摄影/                      ← SKU 摄影主目录，最高频更新
│   ├── by-SKU/                  ← 按 SKU 归档（主路径，所有 SKU 摄影必须在这里至少有一份）
│   │   ├── TEACUP-001/
│   │   │   ├── raw/             ← 原图 / RAW / 高清 JPG（不动 / 不裁切 / 长期保留）
│   │   │   ├── edited/          ← 调色 / 裁切后准备上 Etsy 的成图
│   │   │   └── scene/           ← 场景图（手握、桌面、自然光等）
│   │   ├── TEACUP-002/
│   │   │   └── ...
│   │   └── POT-001/
│   │       └── ...
│   └── shoot-archive/           ← 按拍摄批次的临时目录，用完整理到 by-SKU/
│       └── 2026-05-05_morning-light/
│
├── 2. 视频/                      ← 视频素材主目录（如果店铺暂不做视频，建库时可跳过）
│   ├── by-SKU/                  ← 单 SKU 视频（产品演示、特写、上手）
│   │   └── TEACUP-001/
│   │       ├── raw/             ← 相机 / 手机原始视频，不动
│   │       └── edited/          ← 母版（最高画质，未压缩、未加平台字幕）
│   │           └── _assets/     ← 该视频的附属物：BGM、字幕 SRT、封面图
│   ├── multi-SKU/               ← 跨 SKU 视频（开箱、店铺日常、品牌故事）
│   │   └── 2026-05-05_morning-shop-tour/
│   │       ├── raw/
│   │       └── edited/
│   └── shoot-archive/           ← 按拍摄批次的临时目录（同摄影逻辑）
│
├── 3. 视觉模板/                  ← Etsy listing 配图复用模板（白底加文字、规格图、9 宫格等）
│   ├── listing-cover/           ← Etsy 主图配图模板
│   ├── listing-detail/          ← 细节图模板（带尺寸标注、说明等）
│   ├── shop-banner/             ← Etsy 店铺顶部 banner 模板
│   └── social/                  ← Instagram / 小红书等社媒配图模板
│
├── 4. 字体/                      ← 品牌使用字体的源文件
│   ├── primary/                 ← 主字体（含许可证）
│   └── fallback/                ← 备用字体
│
├── 5. Logo & 品牌标识/           ← 不常更新但要随手能找到
│   ├── primary/                 ← 主 Logo（PNG 透明底 / SVG / 各尺寸）
│   ├── monochrome/              ← 单色版本
│   └── usage-guide.md           ← Logo 使用规范（可选——也可以并入 BRAND.md 视觉段）
│
├── 6. 物料 & 包装/               ← 与产品物理打包相关的素材（包装盒设计、感谢卡、贴纸等）
│   ├── packaging-design/
│   └── thank-you-card/
│
├── 7. 受限/                      ← 客户隐私 / 未授权素材，不公开
│   ├── 客户拍摄/                ← 客户拍的产品照、买家秀（按订单号归档；授权前默认放这里）
│   │   └── ORDER-2026-001/
│   ├── 定制素材/                ← 客户提供的定制参考图（按订单号归档）
│   │   └── ORDER-2026-001/
│   └── 个人参考/                ← 用户个人参考但不发布的素材
│
└── 8. 临时 (待整理)/             ← 任何还没决定归到哪里的素材先放这里
    └── （定期清理，不让它无限膨胀）
```

> **关于 `2. 视频/`**：如果店铺暂时不做视频内容，模式 A 建库时可以跳过这个目录；等真正开始拍视频再补建。但素材索引 Base 的字段保留（"素材类型"里包含"视频原料/视频母版"选项），未来无需改 Base schema。

## 命名规则

- **顶层目录用数字编号**（`1. 摄影` / `2. 视频` / `3. 视觉模板`...）—— 飞书云空间按字母排序，编号让顺序稳定
- 中文目录名 + 英文子目录名混用（顶层中文方便用户识别；子目录英文方便文件路径稳定）
- 受限目录单独放最后，强调权限管控——后续可以单独设访问权限

## 何时新增顶层目录

文件夹有 Base 兜底之后，**新增顶层目录的门槛应该更高**——绝大多数"想新增目录"的诉求其实是"在 Base 加一个多选项"。判断标准：

| 诉求 | 解法 |
|---|---|
| "新春活动专用" | ❌ 不建目录 → ✅ Base 加用途标签 `2026-spring-campaign` 或类似 |
| "客户邮寄的物件照片" | ❌ 不建目录 → 已有 `7. 受限/`；Base 里素材类型勾"客户拍摄" |
| "Pinterest 发布图库" | ❌ 不建目录 → ✅ Base 加用途标签 `Pinterest` |
| "评价素材池" | ❌ 不建目录 → ✅ Base 加用途标签 `评价素材` + 视图筛选 |
| "视频素材"（开始做视频） | ✅ 物理形态完全不同（容量大、需要附属物 BGM/字幕），**值得新增** `2. 视频/` 顶层目录 |
| "音频素材库"（开始做播客） | ✅ 同理，物理形态不同 |

简言之：**物理形态不同才扩骨架；归类维度不同就在 Base 加字段值**。

## 不该归档进资产库的东西

- 用户的私人照片（与店铺无关的家庭照、个人照）
- 已上 Etsy 的最终图（Etsy 后台已经存了，这里只放原始 / 模板 / 工作版本）
- 财务凭证 / 物流单 —— 那是 orders-customers 和 SHOP.md 范畴

## 容量警示

每年回头看一次摄影 raw/ 目录的容量。手作店铺的摄影会**累积得很快**——一年可能上百 GB。考虑：
- 把超过 2 年的旧 raw/ 备份到外部硬盘 + 仅在云空间留 edited/ 成图
- 但本 skill **不主动删除** raw 文件——容量管理由用户人工决策
