# 素材库目录结构

> 在飞书云空间创建 `{店铺名}-素材库` 作为根目录，下设 **6 个一级文件夹**，无任何子目录。

## 核心原则

**文件夹只管来源归属 / 发布阶段，所有细粒度分类交给 Base。**

文件夹回答的唯一问题是"这个素材大致属于哪个来源方向"。SKU 归属、用途槽位、渠道投放、raw/成品阶段、公开授权——全部由[素材索引 Base](asset-index-base-schema.md) 的多选字段和关联字段表达。

绝不要在文件夹层面尝试二次分类：不建 `by-SKU/` 子目录、不建 `raw/edited/` 子目录、不建 `for-pinterest/` 子目录。**一个文件放一个一级文件夹，其余靠 Base 打标签。**

`营销/` 是唯一新增的发布路径桶：它只承接已经面向社交媒体、广告投放、邮件、活动页等营销路径加工过的素材。原始产品摄影、listing 主图、客户 UGC 原始来源仍按来源归属放 `商品/`、`客户/` 等文件夹；是否投 Pinterest / Instagram / 小红书 / 邮件等渠道，仍用 Base 的"用途标签"表达。

### Base 是查素材的唯一入口

用户找素材时走 Base 视图筛选 → 点"文件链接"跳转到云空间文件。**不应该在文件夹里翻找**——文件夹只是存储后端。

未录入 Base 的素材（待处理区的新素材）例外——这些需要在文件夹里浏览，但它们集中在一个固定位置（待处理/），浏览成本可控。

## 目录骨架

```
{店铺名}-素材库/
├── 商品/        ← 所有与商品直接相关的素材：产品摄影、listing 图、产品视频、场景图、shoot-brief
├── 品牌/        ← 品牌层面素材：Logo、字体、视觉模板、品牌故事素材、包装设计稿、店铺 banner
├── 客户/        ← 客户相关素材：买家秀 UGC、定制参考图、客户拍摄（含隐私/受限素材）
├── 工作室/      ← 工作室/工坊环境：制作过程、behind-the-scenes、工具设备、原料展示
├── 营销/        ← 社交媒体、广告、邮件、活动页等营销路径使用的发布素材和派生版本
└── 待处理/      ← 所有新的、未分类素材的入口。分类完成后移入对应文件夹
```

### 各文件夹的典型内容

| 文件夹 | 典型素材 | 说明 |
|---|---|---|
| **商品/** | 产品原图（RAW/JPG）、调色成图、场景图、产品视频、shoot-brief.md | 占比最大的桶；靠 Base 的"关联 SKU"+ "素材类型"字段区分 |
| **品牌/** | Logo PNG/SVG、品牌字体 TTF/OTF、listing 模板 PSD、包装设计稿、店铺 banner、感谢卡设计 | 长期复用、低频更新 |
| **客户/** | 客户拍的产品照、买家秀、定制订单参考图 | 默认受限，"公开授权"字段控制是否可投渠道 |
| **工作室/** | 工坊环境照、制作过程拍摄、工具特写、原料展示 | 品牌故事 / context 类内容的素材来源 |
| **营销/** | Pinterest pin、Instagram post/story/reel 封面、TikTok/Shorts 派生视频、邮件 banner、活动页 KV、广告投放版本 | 已经面向传播路径加工过的成品或母版；具体渠道 / 活动 / SKU 仍靠 Base 标签和关联字段筛 |
| **待处理/** | 刚拍完的整批照片、客户刚发来的图、任何还没分类的素材 | 临时中转，定期清理 |

## 命名规则

文件夹扁平化后，文件名是在文件夹视图里快速识别素材的唯一线索。详细命名公式见 [naming-convention.md](naming-convention.md)。

核心要求：
- **英文 + 数字 + 短横线/下划线**，避免中文路径
- 商品类素材必须在文件名中包含 **SKU 编号**
- 成品（准备使用的）按公式命名；原片可保留相机原始名

## 何时新增一级文件夹

门槛很高。绝大多数"想新增文件夹"的诉求是"在 Base 加一个多选项"。

| 诉求 | 解法 |
|---|---|
| "新春活动专用素材" | ✅ 物理放 `营销/` → Base 加用途标签 `2026-spring-campaign` |
| "Pinterest 发布图库" | ✅ 物理放 `营销/` → Base 筛用途标签 ⊇ Pinterest |
| "评价素材池" | ❌ 不建文件夹 → ✅ Base 筛"客户拍摄 + 已授权" |
| "播客音频"（物理形态全新） | ✅ 值得新增——物理形态完全不同 |

**物理形态或稳定工作流阶段不同才扩骨架；归类维度不同就在 Base 加字段值。**

`营销/` 已经覆盖社媒和增长路径，不要继续按平台建 `Pinterest/`、`Instagram/`、`TikTok/` 子目录；平台差异写进文件名和 Base 标签。

## 不该归档进素材库的东西

- 用户私人照片（与店铺无关）
- 已上 Etsy 的最终图（Etsy 后台已存，这里放原始/工作版本）
- 财务凭证 / 物流单（属于 orders-customers 和 SHOP.md 范畴）

## 容量管理

商品/ 会随时间积累大量文件。建议：
- 超过 2 年的旧原片备份到外部硬盘，云空间仅留成品
- 本 skill **不主动删除**任何素材——容量管理由用户人工决策
