# 素材类型与处理方式

> 本文档帮助 Agent 判断素材属于哪一类：(1) 归到哪个一级文件夹，(2) promote 进 Base 时"素材类型"字段勾哪些选项。

| 类型 | 典型特征 | 归属文件夹 | Base"素材类型"勾选 | 处理建议 |
|---|---|---|---|---|
| 摄影原图（RAW） | `.RAF`、`.CR2`、`.NEF` 等相机 RAW 格式；或大尺寸 `.JPG`（> 5 MB） | `商品/` | `摄影原图` | 长期保留，**不删不改**；命名加日期 + SKU + 拍摄情境 |
| 摄影成图 | 调色裁切后的 `.JPG` / `.PNG`，准备上商品页 / listing 或商品展示用 | `商品/` | `摄影成图` | 命名按用途（listing-cover / detail / scene） |
| 场景图 | 物件在真实场景里（手握、桌面、自然光） | `商品/` | `场景图` | 比成图更"日常感"，体现 BRAND.md 视觉原则的"使用中"语境 |
| 视频原料 | 相机 / 手机原始视频文件，未剪辑 | `商品/` | `视频原料` | 容量大；上传前确认网络条件 |
| 视频母版 | 剪辑完成的最高画质成片，未压缩、未加平台字幕 | `商品/` | `视频母版` | 一份母版可派生多个平台版本 |
| 视觉模板源文件 | `.PSD` / `.AI` / `.SKETCH` / `.FIG` | `品牌/` | `视觉模板` | 模板是**复用资产**，每改一次新建变体而非覆盖 |
| 字体源文件 | `.TTF` / `.OTF` / `.WOFF` | `品牌/` | `字体` | 必须与许可证一起归档 |
| Logo & 品牌标识 | 透明底 PNG / SVG / 各尺寸 | `品牌/` | `Logo` | 必须有透明底版本和单色版本 |
| 包装物料设计 | 包装盒设计稿、感谢卡、贴纸 | `品牌/` | `包装物料` | 与 SHOP.md "物料设置"段呼应 |
| 店铺 Banner | 目标平台店铺顶部 banner / 店铺装修图 | `品牌/` | `视觉模板` | 长期复用 |
| 客户拍摄 / UGC | 客户拍的产品照、买家秀、评价图 | `客户/` | `摄影成图` + `客户拍摄`（多选）| "公开授权"待沟通也可入 Base，但勾渠道用途标签需先改 `已授权` |
| 客户定制参考 | 客户为定制订单提供的参考图 | `客户/` | `客户定制参考` | **不进公开渠道**；订单结束 1 年后可由用户决定是否清理 |
| 工作室环境 | 工坊全景、工作台、工具设备 | `工作室/` | `摄影成图` | 品牌故事 / context 类内容的来源 |
| 制作过程 | 制作步骤拍摄、手工细节 | `工作室/` | `摄影成图` | behind-the-scenes 内容 |
| 原料展示 | 原材料、半成品 | `工作室/` | `摄影成图` | 强调材质 / 工艺的素材 |
| 社媒发布图 | 已按平台比例 / 文案 / 版式加工，准备发 Pinterest、Instagram、小红书等 | `营销/` | `摄影成图` 或 `视觉模板` | 用途标签勾对应渠道；如从 SKU 派生，仍填关联 SKU |
| 营销短视频 / 派生视频 | Reels、TikTok、Shorts、Pinterest video pin、广告投放版本 | `营销/` | `视频母版` | 平台、比例、时长写进文件名；用途标签勾对应渠道 |
| 活动 / 邮件 / 广告素材 | 邮件 banner、活动页 KV、coupon 图、投放素材 | `营销/` | `摄影成图` 或 `视觉模板` | 活动名写进文件名或备注，长期活动用 Base 用途标签筛 |
| 临时素材 | 还没决定归类的 | `待处理/` | （暂不录入 Base） | 定期清理，B2 promote 时移入对应文件夹 |

> 文件夹结构以 [folder-structure.md](folder-structure.md) 为准；本表如有不一致以 folder-structure.md 为锚。

## 多归属是常态

一份素材常常同时是多种类型（如"客户拍摄的茶杯特写成图" = `摄影成图` + `客户拍摄`）。**物理只放一个文件夹**（按来源归属 / 发布阶段决定），Base"素材类型"多选勾全。

例：客户拍的茶杯图：
- 物理位置：`客户/`（客户发来的素材归客户文件夹）
- Base 字段：`素材类型=[摄影成图, 客户拍摄]` + `关联订单=ORDER-XXXX` + `用途标签=[Pinterest, 评价素材]` + `公开授权=已授权`

## 视觉合规自检（B2 promote 时）

对照 BRAND.md `§ 视觉原则` 自检——主动**提醒**而非否决。待处理/ 区不做这个检查（B1 dump 不录 Base），只有 promote 时才执行：

- ❌ 出现视觉禁区元素（红黄金高饱和、龙凤祥云回字纹、Comic Sans 等） → 提醒用户
- ❌ 视觉气质明显偏离品牌 → 同上
- ✅ 符合或基本符合 → 直接给 promote 建议

最终决策权在用户。结论写进 Base 的"BRAND 合规"字段（不通过时把理由写进"备注"）。

## RAW 文件的特殊处理

- 容量大（一张 RAW 可能 30-100 MB）—— 上传飞书云空间前确认用户的网络条件
- RAW promote 到 Base 时素材类型勾"摄影原图"；但通常建议先编辑导出成图再 promote
- RAW 需要专门的工具打开（Lightroom 等），Agent **不要尝试预览或解析 RAW 文件内容**

## 多平台素材的归档策略

同一张图要发多个平台（如 Etsy + Instagram + 小红书），三种比例都需要：

```
商品/
├── TEACUP-001_listing-cover_01.jpg     ← Etsy 主图
├── TEACUP-001_listing-detail_01.jpg

营销/
├── TEACUP-001_social-square_01.jpg     ← Instagram 方图
└── TEACUP-001_social-portrait_01.jpg   ← 小红书竖图
```

电商商品页 / listing 专用图仍放 `商品/`；面向社交媒体、广告、邮件等营销路径加工过的派生版本放 `营销/`。不要为每个平台建子目录——让用途出现在文件名里更易跨平台搜索。Base 用途标签区分渠道归属。

## 与 listing-catalog 的协作

`Assets 素材池` 表的"关联 SKU"字段直接关联到 `Products 商品` 表。通过反向关联视图，SKU 行能看到该 SKU 的所有素材，无需手动维护"照片链接"字段。

如果用户偏好在 `Products 商品` 表上仍保留"照片链接"做快速跳转，每次为某 SKU promote 新成品后提示：

> 「素材已录入 `Assets 素材池` 表，SKU 行通过反向关联可以看到。如果你还希望在 `Products 商品` 表里维护一个跳转链接字段，要不要我请 listing-catalog 更新？」

不要直接调用 listing-catalog —— 由用户决定是否触发跨 skill 协作。
