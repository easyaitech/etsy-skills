# 素材类型与处理方式

> 本文档帮助 Agent 在归档时**判断素材属于哪一类**：(1) 物理放哪条目录，(2) 在素材索引 Base 的"素材类型"字段勾哪些选项。

| 类型 | 典型特征 | 物理位置 | Base"素材类型"勾选 | 处理建议 |
|---|---|---|---|---|
| 摄影原图（RAW） | `.RAF`、`.CR2`、`.NEF` 等相机 RAW 格式；或大尺寸 `.JPG`（> 5 MB） | `1. 摄影/by-SKU/{SKU}/raw/` | `摄影原图` | 长期保留，**不删不改**；命名加日期 + 拍摄情境 |
| 摄影成图 | 调色裁切后的 `.JPG` / `.PNG`，准备上 Etsy 用 | `1. 摄影/by-SKU/{SKU}/edited/` | `摄影成图` | 命名按用途（listing-cover / detail / social）；旧版进 `_archive/` |
| 场景图 | 物件在真实场景里（手握、桌面、自然光） | `1. 摄影/by-SKU/{SKU}/scene/` | `场景图` | 比成图更"日常感"，体现 BRAND.md 视觉原则的"使用中"语境 |
| 视频原料 | 相机 / 手机原始视频文件，未剪辑 | `2. 视频/by-SKU/{SKU}/raw/` 或 `2. 视频/multi-SKU/{batch}/raw/` | `视频原料` | 容量大；上传前确认网络条件 |
| 视频母版 | 剪辑完成的最高画质成片，未压缩、未加平台字幕 | `2. 视频/by-SKU/{SKU}/edited/` 或 `2. 视频/multi-SKU/{batch}/edited/` | `视频母版` | 一份母版可派生多个平台版本；附属物（BGM、SRT、封面）放同目录 `_assets/` |
| 视觉模板源文件 | `.PSD` / `.AI` / `.SKETCH` / `.FIG` | `3. 视觉模板/{purpose}/` | `视觉模板` | 模板是**复用资产**，每改一次新建变体而非覆盖；变体命名带语义（`_minimal-white` `_natural-textured`） |
| 字体源文件 | `.TTF` / `.OTF` / `.WOFF` | `4. 字体/{primary 或 fallback}/` | `字体` | 必须与许可证一起归档；商用字体的合同 / 收据放同目录 |
| Logo & 品牌标识 | 透明底 PNG / SVG / 各尺寸 | `5. Logo & 品牌标识/{primary 或 monochrome}/` | `Logo` | 必须有透明底版本和单色版本；每次改 Logo 都新建变体不覆盖 |
| 包装物料 | 包装盒设计稿、感谢卡、贴纸 | `6. 物料 & 包装/{packaging-design 或 thank-you-card}/` | `包装物料` | 与 SHOP.md "物料设置"段呼应；改版时保留旧版 |
| 客户拍摄 / UGC | 客户拍的产品照、买家秀、评价图 | `7. 受限/客户拍摄/{ORDER-XXXX}/` | `摄影成图` + `客户拍摄`（多选）| Base 里"公开授权"必须填（默认 `待沟通`） |
| 客户定制参考 | 客户为定制订单提供的参考图（家庭照、自定义图案等） | `7. 受限/定制素材/{ORDER-XXXX}/` | `客户定制参考` | **不进公开目录**；订单结束 1 年后可由用户决定是否清理（隐私优先） |
| 临时素材 | 还没决定归类的 | `8. 临时 (待整理)/` | （暂不录入 Base） | 每月清理一次，超过 3 个月没归类的提醒用户决定 |

> 顶层目录编号在 [folder-structure.md](folder-structure.md) 调整后会以最新版本为准；本表如有不一致以 folder-structure.md 为锚。

## 多归属是常态

一份素材常常同时是多种类型（如"客户拍摄的茶杯特写成图" = `摄影成图` + `客户拍摄`）。物理只选一处（按来源决定隐私层级），Base"素材类型"多选勾全。完整示例见 [folder-structure.md § 文件夹的定位](folder-structure.md#文件夹的定位很重要)。

## 归档前的视觉合规自检

对照 BRAND.md `§ 视觉原则` 自检——主动**提醒**而非否决：

- ❌ 出现视觉禁区元素（红黄金高饱和、龙凤祥云回字纹、Comic Sans 等） → 提醒用户："这张图含视觉禁区元素 X，确认要归档吗？"
- ❌ 视觉气质明显偏离品牌 → 同上
- ✅ 符合或基本符合 → 直接给归档建议

最终决策权在用户。结论写进 Base 的"BRAND 合规"字段（不通过时把理由写进"备注"）——后续可在"BRAND 复审"视图集中复盘。

## RAW 文件的特殊处理

- 容量大（一张 RAW 可能 30-100 MB）—— 上传飞书云空间前确认用户的网络条件
- 一般用户不会反复编辑 RAW —— 只在`raw/`目录留一份；编辑过的成图进 `edited/`
- RAW 需要专门的工具打开（Lightroom 等），Agent **不要尝试预览或解析 RAW 文件内容**

## 多平台素材的归档策略

同一张图要发多个平台（Etsy + Instagram + 小红书），三种比例都需要：

```
1. 摄影/by-SKU/TEACUP-001/edited/
├── TEACUP-001_listing-cover_01.jpg     ← Etsy 主图
├── TEACUP-001_listing-detail_01.jpg
├── TEACUP-001_social-square_01.jpg     ← Instagram 方图
└── TEACUP-001_social-portrait_01.jpg   ← 小红书竖图
```

不要为每个平台建子目录（`for-etsy/` `for-ig/`）—— 让用途出现在文件名里更易跨平台搜索。

## 与 listing-catalog 的协作

素材索引 Base 的"关联 SKU"字段直接关联到商品 Base——**商品 Base 通过反向关联视图就能看到该 SKU 的所有素材**，无需手动维护"照片链接"字段。

如果用户偏好商品 Base 上仍保留"照片链接"做快速跳转，每次为某 SKU 归档新摄影后提示：

> 「素材已录入索引 Base，商品 Base 该 SKU 通过反向关联可以看到。如果你还希望在商品 Base 里维护一个跳转链接字段，要不要我请 listing-catalog 更新？」

不要直接调用 listing-catalog —— 由用户决定是否触发跨 skill 协作。
