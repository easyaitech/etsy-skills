# 订单履约 SOP

> 本 SOP 用于新订单 / 待发货订单的全链路执行检查。它不是客服话术；客服话术仍看 `order-handling.md`。

## 适用触发

- 新订单录入订单 Base。
- 订单状态进入 `待发货`。
- 用户问"这单下一步做什么 / 有没有漏什么 / 准备发货了吗"。
- 定制订单需要从确认内容走到制作、出货、跟进评价。

## 执行原则

1. 先查订单 Base、客户 Base、商品 Base，再判断下一步；不要凭聊天记忆推进订单。
2. 对买家的承诺必须来自 `SHOP.md`，尤其是制作周期、发货方式、退换货和定制边界。
3. 需要买家确认的内容必须先出确认材料，再进入不可逆动作。
4. 证据文件进入素材库或可追溯链接；不要只留在聊天里。
5. 优惠券只在用户已提供券码 / 店铺政策允许时发送；没有券码时提醒用户补，不要编造。

## 阶段清单

### 1. 内容确认

目标：确认买家要写什么、做什么、寄什么。

检查：

- 订单号、客户、SKU、数量已录入。
- 定制内容已记录到订单 Base 的 `定制需求`。
- 如涉及文字、姓名、日期、祝福语、颜色、尺寸、收件场景，逐项列给用户确认。
- 需要发给买家的确认消息时，按 `order-handling.md` 的定制咨询规则起草英文消息。

完成条件：

- `买家确认状态 = 已确认`，或用户明确标记不需要买家确认。
- 确认要点写入 `客服记录` 或 `备注`。

### 2. 制作 / 书写

目标：进入生产，但不提前承诺发货。

检查：

- 已确认制作内容。
- 承诺发货日未超期。
- 如制作中发现内容歧义，回到阶段 1，不继续制作。

完成条件：

- 成品完成。
- 订单 Base 更新 `SOP 阶段 = 待出货确认`。

### 3. 出货前拍照确认

目标：让用户确认最终寄出的东西完整、正确。

检查：

- 列出最终会寄出的所有物品：主商品、定制件、礼盒、卡片、说明书、赠品、保护材料。
- 拍摄出货确认照片，至少包含所有物品的合照；定制文字需要可读。
- 照片链接写入 `确认照片链接`。

完成条件：

- 用户确认照片和清单无误。
- `出货清单` 已写入订单 Base。
- `SOP 阶段 = 待打包`。

### 4. 打包并拍视频

目标：证明发货前包装动作完成，降低运输纠纷时的解释成本。

检查：

- 先获得用户确认，再打包。
- 定制 / 高价值 / 易碎订单默认需要打包视频；普通库存单可把 `打包视频状态` 标记为 `不适用`，但不能静默跳过。
- 打包视频应覆盖：商品入盒、保护材料、外箱封装、面单前的最终状态。
- 如需要视频，先写 `打包视频状态 = 必需 / 待补`；完成后把视频链接写入 `打包视频链接`，并更新 `打包视频状态 = 已完成`。

完成条件：

- `打包视频状态 = 已完成` 且 `打包视频链接` 有值，或 `打包视频状态 = 不适用`。
- `SOP 阶段 = 待发货`。

### 5. 发货与物流

目标：订单进入真实物流链路。

检查：

- 运输方式与 `SHOP.md` 对齐。
- 跟踪号已填。
- 发货日期已填。
- 如需要给买家发消息，按 `order-handling.md` 的订单咨询结构起草英文通知。

完成条件：

- `状态 = 已发货`。
- `跟踪号`、`发货日期` 已写入。
- `SOP 阶段 = 待签收跟进`。
- 根据预计签收窗口填写 `签收跟进日期`；无法判断时不要猜，提示用户补物流时效。

### 6. 签收后跟进

目标：引导收货确认、评价和复购，但不要让消息像群发广告。

检查：

- 只有在签收或预计已送达后再发。
- 默认跟进时间可参考 `order-handling.md`：签收 7 天后；如果 `SHOP.md` 或用户另有规则，以用户规则为准。
- 消息必须带一个订单具体细节，避免群发感。
- 优惠券只在用户提供券码 / 店铺允许时提；否则写成"如要附优惠券，请先补券码"。

完成条件：

- 跟进消息草稿整篇给用户确认。
- 用户发出后，摘要写入 `客服记录`。
- `评价跟进状态` 和 `优惠券发送状态` 更新。

## 下一步判断

每次用户问某订单状态时，按下面顺序输出：

1. 当前 `SOP 阶段`。
2. 下一步动作。
3. 缺失证据或字段。
4. 是否需要用户确认。
5. 会写回订单 Base 的字段预览。

如果字段缺失，不要继续编流程。先列缺口。
