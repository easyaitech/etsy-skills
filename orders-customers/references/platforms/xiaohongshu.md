# 小红书订单与客服配置

本文件是小红书订单 / 履约 / 售后 preset。仅在目标销售平台是“小红书 / 小红书店铺 / RED”时读取；不要把这些字段套给 Etsy 或其他平台。preset 契约（一个平台 preset 该定义什么）见 [`platform-presets.md`](platform-presets.md)。

## preset 契约对照

| # | 契约项 | 小红书取值 |
|---|---|---|
| 1 | 平台标识 / 别名 | 小红书 / 小红书店铺 / RED |
| 2 | 买家语言（默认） | 中文 |
| 3 | 履约单元 | 包裹（`packageId`；一个包裹一行 `Orders 订单`） |
| 4 | 平台专属字段组 | `base-schema.md` 的「小红书字段」组（订单 / 包裹 / 商品 / 收件 / 售后） |
| 5 | 订单号 / 买家标识 | `小红书订单 ID` / `小红书包裹 ID`；买家不按 Etsy username 解析 |

| 7 | 消息 / 客服 / 售后入口 | 以 `COMMERCE_PLATFORM.md` 为准；售后通常走小红书平台售后链路 |

依据：
- 小红书 API 应用类目说明：订单管理类应用覆盖订单同步、发货管理、售后管理、订单追踪和异常监控。来源：https://xiaohongshu.apifox.cn/
- 小红书订单基础介绍：订单类型包括内贸订单和跨境订单；订单状态变更为待发货后，可拉取订单详情，订单详情由订单详情、收件人详细地址信息、海关申报信息组成；备货模式订单不需要商家发货。来源：https://xiaohongshu.apifox.cn/doc-2810951
- 小红书 FAQ：订单列表返回的是已支付且可发货订单；订单详情导出前消费者仍可能取消，导出后消费者不能在客户端取消；订单详情导出后退货由小红书客服联系第三方商家处理。来源：https://school.xiaohongshu.com/en/open/faq.html
- 小红书订单接口：订单列表包含 `packageId`、包裹状态、售后状态、取消状态、下单 / 支付 / 发货 / 完成时间、承诺最晚发货时间、收货地区、商家备注、物流模式等。来源：https://xiaohongshu.apifox.cn/api-24196747
- 小红书订单详情接口：订单详情包含 SKU 列表、SKU 数量、SKU 金额、税费、折扣、支付金额、物流单号、快递公司、收件人字段、拆单发货列表、跨境扩展信息等。来源：https://xiaohongshu.apifox.cn/api-24201582
- 小红书订单收件人信息接口：收件人详细地址需要单独接口获取，且订单基础说明明确出于数据安全，操作页面不展示收件人信息，打印快递单时调用。来源：https://xiaohongshu.apifox.cn/api-24202364
- 小红书订单发货接口：发货需要订单 ID、快递单号、快递公司编码 / 名称、发货时间、是否拆单、SKU ID 列表等。来源：https://xiaohongshu.apifox.cn/api-24204499

## 订单数据结构

| 层级 | 含义 | 建议落库字段 |
|---|---|---|
| 订单 / 包裹 | 小红书订单履约单元 | `小红书订单 ID`、`小红书包裹 ID`、`小红书订单类型`、`小红书包裹状态` |
| 商品明细 | 订单里的 SKU 明细 | `小红书 SKU 明细 JSON`，保留 skuId、skuName、数量、金额、税费、折扣 |
| 履约 | 发货、物流、拆单 | `承诺发货日`、`小红书物流模式`、`小红书拆单发货 JSON`、`跟踪号`、`快递公司` |
| 收件人 | 仅在需要发货 / 打单时取 | 不保存完整地址；只保存地区级字段和 `openAddressId` |
| 售后 / 取消 | 取消、退款、退货状态 | `小红书售后状态`、`小红书取消状态`、`小红书售后记录 JSON` |

## 建库字段要求

当 `COMMERCE_PLATFORM.md` 包含小红书平台时，`orders-customers` 模式 A 建 `Orders 订单` / `Customers 客户` 表必须在通用核心字段外加建 `base-schema.md` 的“小红书字段”分组。字段可以先为空，但不要把小红书包裹状态、售后状态、SKU 明细、物流模式塞进备注。

## 录单 / 履约规则

- 新订单录入时先确认平台：小红书订单不要按 Etsy username、Etsy order number 或 Etsy ship-by 字段解析。
- 小红书订单列表只应视为“已支付且可发货”的候选；真正履约前仍要核对订单详情或后台状态。
- 订单详情导出前，消费者可能取消；如果详情接口或后台提示订单不存在 / 已取消，要把 `小红书取消状态` 和备注写清，不要继续推进发货。
- 订单详情导出后，消费者不能在客户端取消；但售后仍可能通过小红书客服流程进入，需单独记录 `小红书售后状态`。
- 收件人完整地址和电话是敏感信息。Base 默认只保存省 / 市 / 区、`openAddressId`、打单证据链接；完整地址留在小红书后台 / 打单系统。
- 发货动作不是本 skill 默认范围。本 skill 只维护 Base、生成核对清单和客服 / 备注草稿；真实发货、改快递单号、售后审核必须由用户或专门平台发布 / ERP skill 执行。

## 客服边界

- 小红书平台内的买家沟通入口、售后协商入口和评价回复入口必须以 `COMMERCE_PLATFORM.md` 配置为准。
- 未配置买家消息渠道时，只输出中文客服草稿和处理建议，不声称可以直接发私信。
- 售后涉及退款、退货、补发时，优先记录事实和待用户确认动作；不要越过小红书售后流程承诺平台外方案。
