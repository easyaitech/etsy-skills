## 当前字段约定

用户明确要求保持精简，已删除 `物料类型`、`适用场景` 两个字段；不要在后续录入时恢复这两个字段。

当前保留字段：

| 字段 | 用法 |
|---|---|
| `物料名称` | 主字段；如 `飞机盒 24×12.5cm`、`中式信封` |
| `店铺名称` | 可空；若已知供应商店铺名可填，必要时可带 Markdown 链接 |
| `商品链接` | 1688 短链 + 口令 + CZ 编号；例如 `https://qr.1688.com/s/... （口令：￥...￥；CZ....）` |
| `合适参数` | 已验证尺寸/规格；淘汰项可写 `尺寸不合适`、`尺寸小了一点点` 等 |
| `状态` | 单选：`主用` / `备用` / `测试中` / `淘汰` |
| `选择理由` | 主用/备用/测试中的选择依据；淘汰项可写 `曾作为...候选来源。` |
| `淘汰原因` | 仅淘汰项填写 |

## 录入口径

- 用户说“可以采购”且无额外限制 → `状态=主用`。
- 用户说“备用” → `状态=备用`，即使“尺寸不合适”，只要用户定义为备用就不要改成淘汰。
- 用户说“淘汰” → `状态=淘汰`，把原因写进 `淘汰原因`。
- 如果用户只给产品名如“飞机盒”，优先把关键尺寸/特征写入 `物料名称`，避免多条同名难区分。
- 对 1688 分享文本，保留：短链、口令、CZ 编号；不要只保存短链。
- 写入前用 `+record-search` 按 `商品链接` 中的短链 token 搜索，避免重复。
- 写入后用 `+record-get` 验证。
- 每次新增/调整采购来源后，同步更新 `~/workspaces/etsy/procurement-management.md` 的修订日志。
